# -Instacart-User-Behavior-Analysis

## 🛒 Instacart Data Analysis
## 📌 Project Description

This project consists of an exploratory data analysis (EDA) of a grocery delivery platform similar to Uber Eats and DoorDash, where customers can place orders and have them delivered to their homes.

The main objective is to explore purchasing patterns, user behavior, and order characteristics using data analysis tools.

## 📂 About the Dataset

The dataset used in this project is a modified version of the original Instacart dataset.

Due to company privacy policies, the following modifications were made:

🔹 The dataset size was reduced to optimize performance and speed up calculations.

🔹 Missing values were intentionally introduced.

🔹 Duplicate records were added for data cleaning practice purposes.

🔹 The original data distributions were preserved to maintain statistical consistency.

These modifications allow for hands-on practice in real-world techniques such as:

Data cleaning
Handling missing values
Removing duplicates
Exploratory data analysis
Data visualization

## 🎯 Project Objectives

Perform data cleaning and preprocessing.
Identify missing values and duplicates.
Analyze purchasing patterns.
Explore order frequency.
Evaluate reorder behavior.
Generate visualizations to support result interpretation.

## 🛠️ Tools Used

Python
Pandas
NumPy
Matplotlib
Jupyter Notebook

## 📊 Analysis Process

Initial dataset exploration.
Identification and treatment of missing values.
Detection and removal of duplicates.
Descriptive analysis.
Data visualization
Conclusions and insights.

## 📈 Expected Results

Through this analysis, the goal is to:
Understand customer purchasing behavior.
Identify the most frequently ordered products.
Analyze order frequency and recurrence patterns.
Extract actionable data-driven insights.

## 📌 Note

This project is intended for educational and data analysis practice purposes.
The dataset has been modified to protect privacy and facilitate learning.

## 🛒 Análisis de Datos – Instacart
📌 Descripción del Proyecto

Este proyecto consiste en el análisis exploratorio de datos (EDA) de una plataforma de entregas de comestibles similar a Uber Eats y DoorDash, donde los clientes pueden registrar pedidos y recibirlos en su domicilio.

El objetivo principal es explorar los patrones de compra, comportamiento de los usuarios y características de los pedidos utilizando herramientas de análisis de datos.

## 📂 Sobre el Conjunto de Datos

El dataset utilizado en este proyecto es una versión modificada del conjunto de datos original de Instacart.

Debido a políticas de privacidad de la empresa, se realizaron las siguientes modificaciones:

🔹 Se redujo el tamaño del conjunto de datos para optimizar el rendimiento y agilizar los cálculos.

🔹 Se introdujeron valores ausentes de manera intencional.

🔹 Se añadieron registros duplicados para fines de práctica en limpieza de datos.

🔹 Se conservaron las distribuciones originales para mantener la coherencia estadística del dataset.

Estas modificaciones permiten practicar técnicas reales de:
Limpieza de datos
Manejo de valores nulos
Eliminación de duplicados
Análisis exploratorio
Visualización de datos

## 🎯 Objetivos del Proyecto

Realizar limpieza y preprocesamiento de datos.
Identificar valores ausentes y duplicados.
Analizar patrones de compra.
Explorar frecuencia de pedidos.
Evaluar comportamiento de recompra.
Generar visualizaciones para facilitar la interpretación de resultados.

## 🛠️ Herramientas Utilizadas

Python
Pandas
NumPy
Jupyter Notebook

## 📊 Proceso de Análisis

Exploración inicial del dataset.
Identificación y tratamiento de valores nulos.
Detección y eliminación de duplicados.
Análisis descriptivo.
Conclusiones e insights.

## 📈 Resultados Esperados

A través de este análisis se busca:
Comprender el comportamiento de compra de los usuarios.
Identificar productos más solicitados.
Analizar frecuencia y recurrencia de pedidos.
Extraer insights accionables basados en datos.
